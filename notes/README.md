# Notes

This directory stores structured infrastructure knowledge.

Notes are not command collections. Each document should explain the essence, relationships, experiments, troubleshooting cases, and future extensions of a topic.

```text
├── 01-Linux/
├── 02-Network/

```

## Rule

Record engineering understanding, not copied documentation.
